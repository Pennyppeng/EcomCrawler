项目结构（建议）

分步实现指南

步骤 1 — 初始化项目与浏览器
- 目标：使用 Playwright 启动一个持久化上下文，保持登录状态以减少重复登录。
- 主要函数：`init_browser(profile_path: str) -> BrowserContext`
- 关键点：使用 `playwright.sync_api.sync_playwright()` 与 `browser_type.launch_persistent_context()`，并把 `user_data_dir` 指向项目下的 `profile/` 文件夹。

示例伪代码：
- 在 `crawler.py` 中：
  - 调用 `init_browser()`，返回 `context` 和 `page`。
  - 在第一次运行时手动登录淘宝并保存配置。

步骤 2 — 实现平滑滚动函数
- 目标：确保页面加载懒加载内容与动态元素。
- 函数签名示例：`def smooth_scroll(page, timeout=30, step=1000):`
- 实现要点：循环执行 `page.evaluate("window.scrollBy(0, step)")`，并使用 `time.sleep()` 或 `page.wait_for_timeout()` 给浏览器时间加载。

步骤 3 — 搜索并翻页（主流程）
- 目标：通过搜索关键词、控制起始页与终止页，遍历各页商品并收集数据。
- 主要流程（在 `crawler.py`）：
  1. 打开 https://www.taobao.com
  2. 定位搜索框并输入关键词（建议使用 `page.locator()` 或 `page.query_selector()`，也可使用 `xpath`）
  3. 点击搜索并等待结果加载
  4. 在每页中调用 `smooth_scroll()`，等待商品数据渲染
  5. 使用 CSS 选择器批量选取商品容器并遍历
  6. 调用 `parsers.parse_product(element)` 获取字段
  7. 存储到内存列表，定期写入磁盘
  8. 翻页：找到“下一页”按钮（推荐使用 copy xpath），点击并等待

注意事项：
- 翻页按钮的父元素或层级常变，遇到失败时用 Chrome DevTools 的 `copy xpath` 获取稳定定位。
- 如果起始页不是 1，先跳转到目标页或使用页码输入跳转逻辑。

步骤 4 — 实现商品解析函数（`parsers.py`）
- 目标：从单个商品 DOM 节点提取标准字段。
- 推荐字段：`title, price, sales, shop_name, shop_location, product_link, shop_link`
- 解析细节：
  - 标题：通过类名或相对路径，例如示例中使用 `.title--ASSt27UY span`（以实际页面为准）
  - 价格：去除换行符和货币符号，转为浮点数
  - 销量：处理含“万”字样的销量字符串，转换为整数（例如“1.2万” -> 12000）
  - 链接：提取 `href` 并补全相对路径

步骤 5 — 数据持久化（`storage.py`）
- 目标：将抓取结果以 CSV/Excel 导出
- 实现建议：使用 pandas.DataFrame，最后调用 `to_excel(..., index=False)` 或 `to_csv()`。
- 同时可实现增量保存：按批次写入避免内存峰值。

步骤 6 — 错误处理与健壮性
- 捕获网络超时、选择器找不到、翻页失败等异常，记录日志并重试（有限次数）。
- 针对翻页失败，尝试：重新定位按钮 xpath、延长等待、模拟鼠标移动再点击。

步骤 7 — 测试与运行
- 在 `main.py` 中实现命令行参数支持：`--keyword`, `--start`, `--end`, `--headless`
- 本地运行示例：

```powershell
python src\main.py --keyword "连衣裙" --start 1 --end 3 --headless False
```

步骤 8 — 登录状态持久化
- 使用 `launch_persistent_context(user_data_dir=profile_dir)` 保存 cookie/session。
- 首次运行时开启有界面模式，手动扫码登录；后续可使用 headless 模式运行。

调试与定位 XPath 的方法（摘录自需求）
- 打开 Chrome，按 F12，进入“元素”面板
- 使用右上角的元素选择器，找到“下一页”时的 DOM 元素
- 右键 -> Copy -> Copy XPath
- 在脚本中使用该 XPath 定位并操作

常见问题与解决办法（摘录并扩展）
- 无法顺利翻页：父元素或按钮类名常变，优先使用 Copy XPath；若仍失败，尝试等待更多时间或模拟滚动/鼠标悬停再点击
- 起始页不是 1 报错：确认页码输入或翻页逻辑是否针对当前页码做了约束
- 每次需重新登录：使用 `launch_persistent_context` 保存登录状态
- 页面自动滚动不灵敏：Playwright 通常比 Selenium 更稳定，优先使用 Playwright
